As Classes Esperadas no POOBank:

Com base no enunciado, o UML ideal deve conter:

  1. Cliente
    • Atributos: nome, CPF, endereço, lista de contas
    • Métodos: registrar chave PIX, listar contas
  2. Conta (classe abstrata)
    • Atributos: número, saldo (private), titular (Cliente), lista de transações
    • Métodos: depositar(valor), sacar(valor), transferir(valor, contaDestino), toJSON()
  3. ContaCorrente (herda de Conta)
    • Atributos: limiteChequeEspecial
    • Métodos: sobrescrever sacar() se necessário
  4. ContaPoupanca (herda de Conta)
    • Atributos: taxaJuros
    • Métodos: renderJuros()
  5. Transacao
    • Atributos: tipo (enum), valor, data, descrição
    • Métodos: estático
  6. PIX
    • Atributos: chave (email, CPF), conta associada
    • Métodos: buscar conta por chave
